# Short Link Service

Сервис сокращения ссылок на Laravel с поддержкой авторизации пользователей и логированием переходов.

---

## Возможности

- генерация short-link из латиницы и цифр
- сохранение соответствия `long_link -> short_link` в БД
- редирект по short-коду
- логирование метаданных визита (`IP`, `User-Agent`, `Timestamp`)
- привязка ссылок к пользователям (One To Many: `User -> Links`)

---

## Зачем нужна авторизация?

Авторизация реализована в рамках требования по привязке сокращаемых ссылок к пользователям.

Это позволяет:
- определить владельца каждой ссылки
- связать пользователя и его ссылки по принципу **One To Many**
- в дальнейшем разграничивать доступ к ссылкам

В рамках данного проекта авторизация используется как надёжный способ идентификации владельца ссылки.

---

## Роуты

### Auth
- `GET /auth` — страница входа
- `POST /auth/login` — логин пользователя

### Создание short-link (требуется авторизация)
- `POST /shorten` — принимает `url` и возвращает страницу с short-ссылкой

### Редирект
- `GET /{code}` — редирект на `long_link` с записью визита (`ip`, `user_agent`, `visited_at`)

---

## Как работает генерация кода

- длина short-кода задаётся в `App\Domain\Entities\Enum\ShortCodeLength` (`SHORT = 6`)
- используются символы:
    - `0-9`
    - `a-z`
    - `A-Z`
- генерация происходит через `random_int()`
- при коллизии (дубликат в БД) происходит повторная генерация через обработку `DuplicateShortLinkException`

## Архитектура (Clean Architecture / SOLID)

Проект построен с разделением на слои:

- `app/Presentation/` — контроллеры, `FormRequest`, входные DTO
- `app/Application/` — use cases, сервисы, DTO
- `app/Infrastructure/` — репозитории, Eloquent-модели
- `app/Domain/` — доменные enum и исключения

---

## База данных

### Таблицы

- `users`
- `links`
    - `short_link` (unique)
    - `long_link`
    - `user_id`
- `link_visits`
    - `link_id`
    - `ip`
    - `user_agent`
    - `visited_at`

---

## Связи

- один пользователь → много ссылок (`users hasMany links`)
- одна ссылка → один пользователь (`links belongsTo users`)
- одна ссылка → много посещений (`links hasMany link_visits`)

---

## Особенности реализации

- short-код состоит только из латиницы и цифр
- уникальность short-link обеспечивается на уровне БД
- при переходе по ссылке сохраняется информация о визите
- создание short-link доступно только авторизованным пользователям
